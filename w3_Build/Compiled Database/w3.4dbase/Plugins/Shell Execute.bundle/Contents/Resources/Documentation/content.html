Shell Execute

Plugin Identifier: 10010

Summary

This plugin is a cross platform tool for launching other applications from 4D or finding out if a certain application is already running. It is intended to be a substitute for the deprecated AP ShellExecute plugin command, offering a simplified interface compared to the more advanced LAUNCH EXTERNAL PROCESS command.

Universal

SE GET PROCESS LIST

SE GET PROCESS LIST (names; paths; numbers)

names (ARRAY)

Process names.

paths (ARRAY)

Process paths.

numbers (ARRAY)

Process numbers.

Discussion

The receiving 3 arrays must be declared appropriately prior to call. On Windows the CreateToolhelp32Snapshot function is used. On Mac, the NSWorkspace class is used.

SE OPEN WITH APPLICATION

SE OPEN WITH APPLICATION (document; application)

document (TEXT)

The document path.

application (TEXT)

The application path.

Discussion

Launches the specified application and instructs it to open the specified document. On Windows, the ShellExecute function is called with the open command. On Mac, the NSWorkspace class is used and the second argument can be either the full path or name of the application.

SE PRINT WITH APPLICATION

SE PRINT WITH APPLICATION (document; application)

document (TEXT)

The document path.

application (TEXT)

The application path.

Discussion

Launches the specified application and instructs it to print the specified document. The called application must be able to respond to the print command.

SE find best application

application:=SE find best application (document)

document (TEXT)

The path of the document to investigate.

application (TEXT)

The path of the application suited to open the document.

Discussion

On Windows, the FindExecutable function is used, which may or may not return a result. On Mac, LSGetApplicationForInfo is used.
